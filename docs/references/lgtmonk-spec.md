# LGTMonk 要件定義書

## 1. プロダクト概要

### 1.1 プロダクト名

**LGTMonk**（LGTM の僧侶）

### 1.2 コンセプト

GitHub のプルリクエストに貼る LGTM 画像を、手軽に生成・検索・コピーできる Web アプリケーション。コードレビューをポップに盛り上げるための"お坊さん"的存在として、いつでもサッと LGTM 画像を差し出す。

### 1.3 背景と課題

| # | 課題 | 解決方針 |
|---|------|----------|
| 1 | コードレビューを盛り上げたいが、毎回同じ LGTM 画像では飽きる | ランダム取得機能で毎回異なる画像を提供する |
| 2 | テンション感を伝えるために自分で選んだ画像を使いたいときもある | キーワード検索・画像アップロードによるカスタム生成を提供する |
| 3 | 既存ツール（LGTMoon 等）が重い・広告が多い・PC 前提で使いづらい | 広告なし・軽量な Web アプリを構築する |
| 4 | 今後スマホで開発する機会が増える見込み | レスポンシブデザインでスマホでもストレスなく操作できるようにする |

### 1.4 ターゲットユーザー

GitHub でコードレビューを日常的に行うエンジニア（個人・チーム問わず）。

## 2. 機能要件

### 2.1 機能一覧

| ID | 機能名 | 認証 | 概要 |
|----|--------|------|------|
| F-01 | LGTM 画像一覧表示 | 不要 | 生成済みの LGTM 画像を「最新順」「ランダム」「お気に入り」のタブ/フィルタで閲覧できる |
| F-02 | 画像コピー | 不要 | 画像をクリックすると、画像 URL および Markdown 埋め込み用テキストをクリップボードにコピーできる |
| F-03 | LGTM 画像生成（アップロード） | 不要 | ユーザーが任意の画像をアップロードし、LGTM テキストを合成した画像を生成する |
| F-04 | LGTM 画像生成（Google 画像検索） | 不要 | キーワードで Google 画像検索を行い、選択した画像に LGTM テキストを合成する。検索キーワードも保存する |
| F-05 | LGTM 画像検索 | 不要 | キーワードで既存の LGTM 画像を検索し、一致するものがなければ生成フロー（F-03 / F-04）へ誘導する |
| F-06 | お気に入り登録（未ログイン） | 不要 | お気に入りした画像の ID をブラウザの localStorage に保存する |
| F-07 | ユーザー認証 | — | Firebase Auth によるログイン/ログアウト機能を提供する |
| F-08 | データ永続化（ログイン時） | 必要 | ログインユーザーは「自分が生成した画像」「お気に入り画像」を Firestore に永続保存できる |
| F-09 | お気に入りマイグレーション | 必要 | 未ログイン状態で localStorage に保存していたお気に入りを、ログイン時に Firestore へ自動マイグレーションする |

### 2.2 画面構成

```
/                  … 画像一覧（トップページ）。タブやモーダルで画像検索、画像生成機能も搭載
/mypage            … マイページ（生成履歴 / お気に入り一覧）※要ログイン
/login             - ログイン
/signup            - 新規登録
```


### 2.3 各機能の詳細

#### F-01: LGTM 画像一覧表示

- 表示モードは「最新」「ランダム」「お気に入り」の 3 種類をタブで切り替える。
- 画像はグリッドレイアウトで表示し、追加読み込みはなし。表示内容を変えたい場合はリロードする。
- 画像をクリックするとモーダルで詳細を表示。画像サムネイル・キーワード（あれば）・コピーボタンが表示される。

#### F-02: 画像コピー

- 画像クリック時にモーダルまたはポップオーバーを表示し、以下をワンクリックでコピーできるようにする。
  - 画像 URL（例: `https://storage.googleapis.com/.../lgtm_xxx.webp`）
  - Markdown 埋め込みテキスト（例: `![LGTM](https://storage.googleapis.com/.../lgtm_xxx.webp)`）
- コピー成功時はコピーボタンを「コピーしました」に変更する
- コピー回数を Firestore の `copiedCount` フィールドにインクリメントする。

#### F-03: LGTM 画像生成（アップロード）

- ユーザーがローカルから画像ファイル（JPEG / PNG / WebP / GIF）をアップロードする。
- 画像をbase64にエンコードしてCloud Functionsの画像生成APIに送る。
- サーバーサイド（Cloud Functions + napi-rs/sharp）で画像に「LGTM」テキストを合成し、WebP 形式で Firebase Storage に保存する。
- 生成した画像のURLをフロントエンドに返却する。
- 生成結果をプレビュー表示し、そのままコピー操作へ遷移できるようにする。

#### F-04: LGTM 画像生成（Google 画像検索）

- キーワード入力 → フロントエンドでGoogle Custom Search API（画像検索）で候補画像を取得 → ユーザーが画像を選択 → LGTM テキストを合成して保存する。
- 検索に使用したキーワードを Firestore の `keyword` フィールドに保存する。

#### F-05: LGTM 画像検索

- キーワード入力で Firestore の `keyword` フィールドを検索し、一致する LGTM 画像を一覧表示する。
- 該当画像が 0 件の場合、「この画像はまだありません。生成しますか？」と生成画面（F-03 / F-04）へ誘導する。

#### F-06: お気に入り登録（未ログイン）

- 各画像カードにお気に入りボタン（♡）を設置する。
- 未ログイン時は画像 ID の配列を localStorage に保存し、一覧画面の「お気に入り」タブで表示する。

#### F-07: ユーザー認証

- Firebase Auth を使用し、Google アカウントによるソーシャルログインを提供する。
- ログイン状態はヘッダーのアバター/ログインボタンで視覚的にフィードバックする。

#### F-08: データ永続化（ログイン時）

- ログインユーザーが画像を生成した場合、`users/{uid}/generates` サブコレクションに記録する。
- ログインユーザーがお気に入りした場合、`users/{uid}/favorites` サブコレクションに記録する。

#### F-09: お気に入りマイグレーション

- ログイン完了時に localStorage のお気に入りデータを検出し、Firestore の `users/{uid}/favorites` に一括書き込みする。
- マイグレーション完了後、localStorage のお気に入りデータをクリアする。

## 3. 非機能要件

### 3.1 パフォーマンス

- トップページの初回表示は 2 秒以内（LCP 基準）を目標とする。
- 画像生成処理は 5 秒以内にレスポンスを返す。
- 画像は WebP フォーマットで配信す。サムネイル用などにリサイズ済みの軽量版は生成しない（オブジェクトストレージの容量節約のため）。

### 3.2 レスポンシブデザイン

- モバイルファーストで設計し、スマートフォン（375px〜）・タブレット・デスクトップの各ブレークポイントに対応する。
- タッチ操作でのコピー・お気に入り操作がスムーズに行えるようにする。

### 3.3 セキュリティ

- 画像アップロード時にファイルタイプ・サイズのバリデーションをクライアント/サーバー双方で実施する（最大ファイルサイズ: 10MB）。
- Firebase Security Rules により、認証済みユーザーのみが自身のサブコレクションに書き込めるよう制御する。

### 3.4 可用性・運用

- Firebase のマネージドサービスを活用し、インフラ運用コストを最小化する。
- Cloud Functions のコールドスタート対策はしない。インフラ費節約のため、最小インスタンス数の設定は0にする。

| レイヤー | 技術 |

| フレームワーク | TanStack Start (TypeScript) |

| サーバーレス関数 | Cloud Functions for Firebase |

| 外部 API | Google Custom Search API（画像検索） |

## 5. データベース設計

### 5.1 コレクション構成

```
firestore/
├── images/                          # LGTM 画像コレクション
│   └── {imageId}/
│       ├── imageUrl: string         # Firebase Storage 上の画像 URL
│       ├── keyword: string          # 生成時の検索キーワード（アップロード時は空文字）
│       ├── impressionCount: number  # 表示回数
│       ├── copiedCount: number      # コピーされた回数
│       ├── createdBy: string | null # 生成者の uid（未ログイン時は null）
│       ├── createdAt: timestamp     # 生成日時
│       └── updatedAt: timestamp     # 更新日時
│
└── users/                           # ユーザーコレクション
    └── {uid}/
        ├── email: string            # メールアドレス
        ├── displayName: string      # 表示名
        ├── createdAt: timestamp     # 登録日時
        │
        ├── generates/               # 生成した画像（サブコレクション）
        │   └── {imageId}/
        │       ├── imageUrl: string # Firebase Storage 上の画像 URL
        │       └── createdAt: timestamp
        │
        └── favorites/               # お気に入り画像（サブコレクション）
            └── {imageId}/
                ├── imageUrl: string # Firebase Storage 上の画像 URL
                └── createdAt: timestamp
```

### 5.2 インデックス設計

| コレクション | フィールド | 用途 |
|------------|-----------|------|
| images | createdAt DESC | 最新順の一覧表示 |

## 7. ユーザーフロー

### 7.1 画像をすぐにコピーしたい場合

1. トップページにアクセスする。
2. 「最新」または「ランダム」タブで画像一覧を閲覧する。
3. 気に入った画像をクリックする。
4. 「Markdown をコピー」ボタンを押す。
5. GitHub のプルリクエストのコメント欄にペーストする。

### 7.2 画像を自分で生成したい場合

1. トップページにアクセスする。
2. 生成タブを表示する。
3. 画像をアップロードする、またはキーワードで Google 画像検索を行い候補から選択する。
4. 「生成」ボタンを押して保存する。
5. 生成された画像の Markdown をコピーする。

### 7.3 キーワードで既存画像を探したい場合

1. 検索画面にアクセスする。
2. キーワード（例: 「猫」「拍手」「最高」）を入力して検索する。
3. 該当する画像があればクリックしてコピーする。
4. 該当する画像がなければ、生成フローへ遷移する。

## 8. 将来的な拡張計画

### 8.1 Chrome 拡張機能

- ブラウザ拡張として動作し、GitHub のプルリクエスト画面上からワンクリックでランダムな LGTM 画像を取得・貼り付けできるようにする。
- Web アプリを開かずに完結するため、レビュー業務の効率が大幅に向上する。
- 拡張から LGTMonk API を呼び出し、画像 URL を取得する設計とする。

### 8.2 その他の検討事項

- GitHub 認証対応（GitHub アカウントでのログイン）。
- 画像へのリアクション機能（👍 / 🔥 等）によるランキング。
- 不適切画像のモデレーション（自動検出 + 通報機能）。
- LGTM テキストのフォント・位置・色のカスタマイズ機能。
