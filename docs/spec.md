# NoyakuNote 要件定義書（仮称）

## 1. プロダクト概要

### 1.1 プロダクト名

**NoyakuNote**（仮称）

### 1.2 コンセプト

農家向けの農薬管理 Web アプリケーション。FAMIC（農林水産消費安全技術センター）の農薬登録情報を活用した農薬検索、散布記録の管理、希釈計算、PDF 帳票出力を提供し、農薬管理業務をデジタル化する。

### 1.3 背景と課題

| # | 課題 | 解決方針 |
|---|------|----------|
| 1 | どの農薬がどの作物に登録されているか調べるのに時間がかかる | FAMIC の農薬登録情報を自前 DB に取り込み、作物名で検索すると該当農薬を一覧表示する |
| 2 | 農薬の散布記録を紙や Excel で管理しており、散逸しやすい | デジタルで散布履歴を記録・管理する機能を提供する |
| 3 | 希釈倍率の計算を毎回手計算しており、ミスのリスクがある | 水量と倍率から必要農薬量を自動計算する機能を提供する |
| 4 | 直売所・市場出荷時に農薬散布記録の提出資料を作成するのが手間 | 散布履歴を PDF 帳票として出力する機能を提供する |

### 1.4 ターゲットユーザー

直売所・市場への出荷を行う個人農家、小規模農業法人。

### 1.5 もらった要望の原文

```
アプリに求めていること

・ルーラル電子図書館(ウェブサイト)
の登録農薬検索コーナーから農薬の情報を引っ張ってくれる機能が欲しい。

具体的には: ｢大根｣と検索をかけたら
大根の病気と害虫に登録のある農薬が一覧でばっと表示されるようにしてほしい。

・使った農薬の日付と倍率、量、使った圃場(畑の場所)を履歴として残す機能

・農薬の希釈倍率の計算機能

(使う水の量と倍率を入力すると農薬を何ml使えば良いのか出してくれる機能。＊農薬は粉と液体があるので、ml、L、g、kgと単位を分けてくれると有り難い！)


・上記の記録をPDF ファイルで出力出来るようにして欲しい(直売や市場に出す資料として使いたい) 
```

### 2.3 各機能の詳細

#### F-01: ユーザー認証

- Firebase Authentication を使用し、電話番号（SMS）認証によるログインを提供する。
- ログインフロー:
  1. ユーザーが電話番号（日本の携帯番号）を入力する。
  2. Firebase が SMS で 6 桁の認証コードを送信する。
  3. ユーザーが認証コードを入力し、検証が成功するとログインが完了する。
- 不正利用防止のため、reCAPTCHA による検証を導入する（Firebase の invisible reCAPTCHA を使用）。
- 未ログイン状態ではログイン画面のみ表示する。
- ログイン成功後、ダッシュボードへ遷移する。
- ログイン状態はヘッダーの電話番号表示/ログインボタンで視覚的にフィードバックする。

#### F-02: 農薬検索

- テキスト入力で作物名（例:「大根」「トマト」）を入力して検索する。
- 検索結果として、その作物の病害虫に登録のある農薬を一覧表示する。
- 各農薬について、農薬名・有効成分・対象病害虫・希釈倍率・使用回数制限を表示する。
- 検索結果から農薬を選択すると、散布記録の登録画面へ遷移できる（農薬情報を引き継ぎ）。

**農薬データの取得・格納設計:**

FAMIC（農林水産消費安全技術センター）が公開している農薬登録情報の CSV ダウンロードサービスを利用し、PostgreSQL に農薬マスターデータを格納する。Firestore は検索性（全文検索・部分一致・JOIN）に制約があるため、農薬マスターデータには PostgreSQL を採用する。

- **データソース**: https://www.acis.famic.go.jp/ddata/index2.htm
- **ファイル構成**:
  - 登録基本部（約 110KB）: 登録番号、用途、農薬の種類名、農薬の名称、会社名略称
  - 登録適用部1（約 680KB）: 登録番号 52〜22553 の適用情報
  - 登録適用部2（約 460KB）: 登録番号 22554〜 の適用情報
- **適用部のカラム**: 登録番号、作物名、適用病害虫雑草名、使用目的、希釈倍数・使用量、使用時期、使用液量、本剤の使用回数、使用方法、適用場所、適用土壌、混合数、有効成分を含む農薬の総使用回数
- **更新頻度**: 新規・変更登録は翌々日、失効情報は月初に反映

**データ取込フロー:**
1. Cloud Functions の定期バッチ（Cloud Scheduler）で FAMIC の CSV（ZIP）をダウンロードする。
2. ZIP を展開し、基本部・適用部の CSV をパースする。
3. 基本部を `pesticides` テーブルに UPSERT する。
4. 適用部1 + 適用部2 を `pesticide_applications` テーブルに UPSERT する。
5. 実行頻度は週 1 回程度を想定（FAMIC の更新頻度に合わせて調整可能）。

**検索フロー:**
1. フロントエンドから Cloud Functions の `searchPesticides`（onCall）を呼び出す。
2. Cloud Functions が PostgreSQL に対して作物名で SQL クエリを実行する。
3. `pesticide_applications` テーブルを作物名で検索し、`pesticides` テーブルと JOIN して結果を返却する。

**コスト: 無料**（FAMIC CSV は無償公開。外部 API の契約不要。PostgreSQL は Cloud SQL 等を使用）

#### F-03: 農薬詳細表示

- 農薬名、農薬登録番号、有効成分、有効成分濃度、剤型（液剤・粉剤・粒剤など）を表示する。
- 対象作物一覧、対象病害虫一覧、希釈倍率/使用量、使用時期、使用回数制限、使用方法を表示する。
- 「この農薬で記録する」ボタンから散布記録登録画面へ遷移できる。

#### F-04: 圃場管理

- 圃場の一覧表示・新規登録・編集・削除が可能。
- 入力項目: 圃場名（必須）、栽培作物（任意）、面積（任意、単位: a アール）、備考（任意）。
- 散布記録登録時に圃場をセレクトボックスから選択するため、事前登録が必要。
- 圃場に紐づく散布記録がある場合、削除時に警告ダイアログを表示する。

#### F-05: 散布記録の登録

- 入力項目:
  - 散布日（必須、未来日不可）
  - 農薬名（必須、農薬検索から選択 or 手動入力）
  - 対象作物（必須）
  - 対象病害虫（任意）
  - 圃場（必須、登録済み圃場から選択）
  - 希釈倍率（必須）
  - 散布量と単位（必須、ml / L / g / kg）
  - 農薬使用量と単位（必須、ml / L / g / kg、希釈計算機から自動入力可）
  - 備考（任意）
- 農薬検索画面から遷移した場合、農薬名・希釈倍率が自動入力される。
- 希釈計算機との連携が可能（計算結果を散布記録に反映）。

#### F-06: 散布記録の一覧・詳細表示

- 散布日の降順（最新順）で一覧表示する。
- フィルタリング条件: 期間（開始日〜終了日）、圃場、作物名。
- 各レコードは散布日・農薬名・対象作物・圃場名・散布量を表示する。
- レコードをタップすると詳細画面に遷移し、全項目を確認・編集・削除できる。

#### F-07: 希釈計算機

- 剤型選択（液剤 / 粉剤・水和剤）で結果の単位を自動切替する。
- 入力: 水量（L）、希釈倍率。
- 出力: 必要農薬量（液剤は ml、粉剤は g）。

**計算式:**

```
農薬量 = 水量 (L) × 1000 ÷ 希釈倍率
```

- 液剤の例: 水 100L、1000 倍希釈 → 100 × 1000 ÷ 1000 = **100 ml**
- 粉剤の例: 水 100L、2000 倍希釈 → 100 × 1000 ÷ 2000 = **50 g**

- 単位変換ボタンを提供する（ml ⇔ L、g ⇔ kg）。
- 計算結果を散布記録登録画面に反映するボタンを提供する。

#### F-08: PDF 出力

- 出力条件の選択: 対象期間（開始日〜終了日）、対象圃場（全て or 個別選択）。
- 帳票ヘッダー: 生産者名（ユーザー登録名）、出力日、対象期間、対象圃場。
- 帳票本体（テーブル形式）:

| No. | 散布日 | 圃場名 | 対象作物 | 農薬名 | 希釈倍率 | 散布量 | 農薬使用量 | 対象病害虫 | 備考 |
|-----|--------|--------|---------|--------|---------|--------|-----------|-----------|------|

- クライアントサイドで PDF を生成する（jspdf + jspdf-autotable）。
- 日本語フォント対応が必要（NotoSansJP 等）。

#### F-09: サブスクリプション決済

Firebase Extension「Run Payments with Stripe」を使用し、Stripe によるサブスクリプション決済を実装する。

**料金プラン:**

| プラン | 料金（税込） | 備考 |
|--------|-------------|------|
| 月額プラン | 2,980 円/月 | — |
| 年額プラン | 29,800 円/年 | 月額換算 約2,483 円（2 ヶ月分お得） |

**無料トライアル:**
- 新規登録ユーザーには 14 日間の無料トライアル期間を提供する。
- トライアル期間中はすべての有料機能（農薬検索・農薬詳細表示・PDF 出力）を利用可能とする。
- トライアル終了後、有料プランに加入しない場合は無料機能のみ利用可能となる。

**無料 / 有料の機能制限:**

| 機能 | 無料ユーザー | 有料ユーザー（トライアル含む） |
|------|-------------|-------------------------------|
| ユーザー認証 | ✅ | ✅ |
| 圃場管理 | ✅ | ✅ |
| 散布記録の登録・一覧・詳細 | ✅ | ✅ |
| 希釈計算機 | ✅ | ✅ |
| 農薬検索・詳細表示 | ❌ | ✅ |
| PDF 出力 | ❌ | ✅ |
| サブスクリプション管理 | ✅ | ✅ |

**技術構成:**
- Firebase Extension「Run Payments with Stripe」を使用し、Stripe と Firestore 間のサブスクリプション状態を自動同期する。
- Stripe Checkout を利用してサブスクリプションの申し込み・決済を処理する。
- Stripe Customer Portal を利用してプラン変更・解約・支払い方法の変更をユーザー自身で行えるようにする。
- サブスクリプションの状態は Firestore 上の各ユーザードキュメント配下に自動的に同期される（Extension が管理）。
- フロントエンドではサブスクリプションの状態を参照し、有料機能へのアクセス制御を行う。

**画面構成への追加:**
- `/settings` 画面にサブスクリプション管理セクションを追加する（現在のプラン表示、プラン変更、解約への導線）。
- 無料ユーザーが有料機能にアクセスした際に、プランへの加入を促すアップグレード画面を表示する。

## 3. 非機能要件

### 3.1 パフォーマンス

- トップページの初回表示は 3 秒以内（LCP 基準）を目標とする。
- 農薬検索結果の表示は 2 秒以内を目標とする。
- PDF 生成は 5 秒以内にダウンロード開始を目標とする。

### 3.2 レスポンシブデザイン

- モバイルファーストで設計し、スマートフォン（375px〜）・タブレット・デスクトップに対応する。
- 農家が圃場でスマートフォンから記録を入力するユースケースを重視し、タッチ操作での入力が容易な設計とする。

### 3.3 セキュリティ

- Firebase Authentication による認証を必須とする。
- Firestore セキュリティルールにより、ユーザーは自身のデータのみアクセス可能とする。
- PostgreSQL への接続は Cloud Functions からのみ行い、クライアントから直接接続しない。
- FAMIC CSV の取込処理は Cloud Functions 上で実行し、クライアントからは直接アクセスしない。

### 3.4 可用性・運用

- Firebase のマネージドサービスを活用し、インフラ運用コストを最小化する。
- Cloud Functions のコールドスタート対策はしない。最小インスタンス数は 0 とする。
- 農薬マスターデータの更新は Cloud Scheduler で定期実行する（週 1 回程度）。

## 5. データベース設計

※ 詳細な DB 設計は別途記載する。ここでは構成の概要のみ示す。

### 5.1 Firestore（ユーザーデータ）

```
Firestore
├── users: ユーザーコレクション
│   ├── createdAt: Timestamp 作成日時
│   ├── email: String メールアドレス
│   └── updatedAt: Timestamp 更新日時
│
├── fields: 圃場コレクション
│   ├── createdAt: Timestamp 作成日時
│   ├── name: String 圃場名
│   ├── updatedAt: Timestamp 更新日時
│   └── userId: String ユーザーID
│
├── sprays: 農薬の散布記録コレクション
│   ├── createdAt: Timestamp 作成日時
│   ├── dilutionRate: Number 希釈倍率
│   ├── fieldId: String 圃場ID
│   ├── fieldName: String 圃場名
│   ├── note: String 備考・メモ
│   ├── pesticideAmount: Number 農薬使用量
│   ├── pesticideName: String 農薬名
│   ├── pesticideUnit: String 農薬の単位（液体or粉末）
│   ├── registrationNumber: String 農薬登録番号
│   ├── sprayAmount: Number 散布量
│   ├── sprayedAt: Timestamp 散布した日付
│   ├── updatedAt: Timestamp 更新日時
│   └── userId: String ユーザーID
│
└── ※ Firebase Extension「Run Payments with Stripe」が自動管理するコレクション:
    ├── users/{userId}/checkout_sessions: Stripe Checkout セッション
    ├── users/{userId}/subscriptions: サブスクリプション状態
    ├── users/{userId}/payments: 支払い履歴
    └── products: Stripe 上の商品・価格情報の同期
```

### 5.2 PostgreSQL（農薬マスターデータ）

農薬マスターデータは検索性を重視し PostgreSQL に格納する。Cloud Functions 経由でのみアクセスする。

テーブル定義・マイグレーションは Prisma で管理する。スキーマは `prisma/schema.prisma` に定義する。

**テーブル構成:**
- `pesticides` — 農薬基本情報（登録番号、名称、種類名、用途、会社名）
- `pesticide_applications` — 農薬適用情報（作物名、病害虫名、希釈倍数、使用時期、使用方法、使用回数等）
- `pesticide_applications.crop_name` にインデックスを設定し、作物名検索を高速化する

## 7. ユーザーフロー

### 7.1 農薬を検索して散布記録をつける

1. ログイン後、ダッシュボードの「農薬検索」をタップする。
2. 作物名（例:「大根」）を入力して検索する。
3. 検索結果から使用する農薬を見つけ、「この農薬で記録する」をタップする。
4. 散布記録登録画面に遷移し、農薬名・希釈倍率が自動入力される。
5. 散布日・圃場・散布量を入力する（必要なら希釈計算機で農薬使用量を計算する）。
6. 「記録する」ボタンで保存する。

### 7.2 希釈計算機を使う

1. 「計算機」画面に遷移する。
2. 剤型（液剤 or 粉剤）を選択する。
3. 水量（L）と希釈倍率を入力する。
4. 必要農薬量が自動計算されて表示される。
5. 散布記録登録画面から遷移した場合、「散布記録に反映する」ボタンで戻る。

### 7.3 散布記録を PDF 出力する

1. 「PDF 出力」画面に遷移する。
2. 対象期間と対象圃場を選択する。
3. 「PDF 出力」ボタンをタップする。
4. PDF がダウンロードされる。

## 8. 農薬データソース

- **データソース**: FAMIC 農薬登録情報 CSV ダウンロード（https://www.acis.famic.go.jp/ddata/index2.htm）
- **コスト**: 無料（外部 API の契約不要）
- **更新頻度**: 新規・変更登録は翌々日、失効情報は月初に FAMIC 側で反映
- **取込頻度**: Cloud Scheduler で週 1 回の定期バッチを実行
- **著作権**: FAMIC に帰属。出典明記のうえ利用する
